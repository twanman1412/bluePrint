Conventions

Conventions are standard ways of doing things in bluePrint. There are very basic principles need to write and understand bluePrint files

File names

A bluePrint file has the file extention .bp. Files without the .bp extention will be ignored by the compiler.

Comments

A comment is text within the code that is ignored by the compiler. They can be used as hints and descriptions in your code.

A single-line comment is started with an # and ends at the end of the line.

A multi-line comment starts with -# and ends with #-.

For example:

# This is a single line comment

-#
This is a multi line comment.
It continues on the line below!
#-

Line endings

Every line in a bluePrint file, except for comments, must end with an ; to indicate the line ending. The only exception to this rule are attributes of classes, which are explained in their respective sections.

# Line ended with ';'
int myInt = 10;

# Line not ended with ';'
int myInt = 10
Syntax Error: ';' expected

Declaration and definition

In blueprint if an object is created is is declared, but not defined. An object is only defined when a value is written to the object. When using an object that has not been defined it will have a value of undefined.

Scope

The scope of an object is the part of the current file in which the object can be used. If an object is referenced outside its scope, it will not be defined ,since it is not part of the current scope, and the program will throw an error.

Code blocks

The current scope can be decreased by declaring a code block using curcly-braces, {}. The scope of objects within the code block is the code block, and the code block is no longer part of the global scope.

# Declaring a code block
{
int number;

# Using the object within scope
int doubled_number = 2 * number;
}

# Using the object outside scope
int doubled_number = 2 * number;
Error: variable number is not defined

Variables

Variables are objects that have a name and a value. Variable can store any value that is the same type as their type, and can be changed at any time. When using a variable you can use the name of the variable, and the program will know to use the value stored in the variable.

Numbers

The number type variables are variables that hold numbers. Since there are many types and ways to store numbers in a computer, the number types have been split up into several different number types, each having their own usage.

Integers

An integer in BluePrint is a 32-bit integer. This means that an integer can be a whole number between -2^31 and 2^31 - 1.

An integer can be declared and defined as following:

# Declaring an int
int myInt;

# Declaring and defining and int
int myInt = 10;

# Declaring an Integer
Integer myInt;

# Declaring and defining and Integer
Integer myInt = 10;

The difference between using int and Integer is explained in primitive and non-primitive types (tba).

Floating point numbers

A floating point number in BluePrint is a 32-bit floating point number, following the IEEE 754 standard. This means that a float can be a number between -3.40 * 10^38 and 3.40 * 10^38, and with ~7 decimal places.

An floating point number can be declared and defined as following:

# Declaring a float
float myFloat;

# Declaring and defining a float
float myFloat = 10.124356;

# Declaring a Float
Float myFloat;

# Declaring and defining a Float
Float myFloat = 10.124356;

The difference between using float and Float is explained in primitive and non-primitive types (tba).

Double precision numbers

A double precision number in BluePrint is a 64-bit floating point number, following the IEEE 754 standard. This means that a double can be a number between -1.80 * 10^308 and 1.80 * 10^308, and with ~15 decimal places.

An double precision number can be declared and defined as following:

# Declaring a double
double myDouble;

# Declaring and defining a double
double myDouble = 10.1234567890;

# Declaring a Double
Double myDouble;

# Declaring and defining a Double
Double myDouble = 10.1234567890;

The difference between using double and Double is explained in primitive and non-primitive types (tba).

Long integers

A long integer is a 64-bit integer. This means that a long integer can be a whole number between -2^63 and 2^63 - 1.

A long integer can be declared and defined as following:

# Declaring a long
long myLong;

# Declaring and defining a long
long myLong = 12345678901234567890;

# Declaring a Long
Long myLong;

# Declaring and defining a Long
Long myLong = 12345678901234567890;

The difference between using long and Long is explained in primitive and non-primitive types (tba).

Characters

A character is a 8-bit integer. This means that a char can be a whole number between -128 and 127. The numbers 0 to 127 directly map to the ASCII characters. Characters within that range will be used as their ASCII character, and can be assigned as their ASCII character.

A character can be declared and defined as following:

# Declaring a char
char myChar;

# Declaring and defining a char
char myChar = 65;

# Declaring and defining a char using the ASCII character
char myChar = 'a';

# Declaring a Char
Char myChar;

# Declaring and defining a Char
Char myChar = 65;

# Declaring and defining a Char using the ASCII character
Char myChar = 'a';

The difference between using char and Char is explained in primitive and non-primitive types (tba).

Booleans

A boolean is a variable that can hold two values, it is either true or it is false.

A boolean can be declard and defined as following:

# Declaring a bool
bool myBool;

# Declaring and defining a bool
bool myBool = true;

Functions and methods

Functions

A function is a block of code that has a name, a return type and parameters. A function can be called from anywhere within the current file, and can use other functions and variables within scope.

A function can be declared as returnType name(arguments);. A function is defined by creating a code block surrounded with curly-braces ({}). Declaration and definiton can also be combined together and done in one go.

-#
The declaration of a function with return type 'int',
name 'double_number' and parameter 'num' of type 'int'.
#-
int double_number(int num);

-#
The definition of a function with return type 'int',
name 'double_number' and parameter 'num' of type 'int'.
#-
int double_number(int num) {

int doubled = 2 * num;
return doubled;
}

A function can then be called from anywhere within scope by its name and by adding the required parameters. A function can also be used to define a variable with the return value.

# Function from previous example
int double_number(int num) {

int doubled = 2 * num;
return doubled;
}

# Defining a variable with the return value of a function
int doubled_number = double_number(5);

Methods

A method is a function that is a member of a class (tba). It must be defined and called either from within the class, or with the class reference.

class MyClass {
# Declaring a method
void myFunction();

# Defining a method within the class
void myFunction() {
# Do something
}
}

# Defining a method with a class reference
MyClass::myFunction() {
# Do something
}

When defining a method using its class reference its footprint will be taken from the class definition, thus the return type does not have to explicitly be stated. Arguments do have to be explicitely stated in order to allow use within the function definition.
