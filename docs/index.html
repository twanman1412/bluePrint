Conventions

File names

A bluePrint file has the file extention '.bp'. Files without the '.bp' extention will be ignored by the compiler.

Comments

A comment is text within the code that is ignored by the compiler. They can be used as hints and descriptions in your code.

A single-line comment is started with an '#' and ends at the end of the line.

A multi-line comment starts with '-#' and ends with '#-'.

For example:

# This is a single line comment

-#
This is a multi line comment.
It continues on the line below!
#-

Line endings

Every line in a bluePrint file, except for comments, must end with an ';' to indicate the line ending. The only exception to this rule are attributes of classes, which are explained in their respective sections.

# Line ended with ';'
int myInt = 10;

# Line not ended with ';'
int myInt = 10
Syntax Error MyBluePrintFile.bp:
4:15 -> ';' expected

Declaration and definition

In blueprint if an object is created is is declared, but not defined. An object is only defined when a value is written to the object. When using an object that has not been defined it will have a value of 'undefined'.

Numbers

Integers

An integer in BluePrint is a 32-bit integer. This means that an integer can be a whole number between -2^31 and 2^31 - 1.

An integer can be declared and defined as following:

# Declaring an int
int myInt;

# Declaring and defining and int
int myInt = 10;

# Declaring an Integer
Integer myInt;

# Declaring and defining and Integer
Integer myInt = 10;

The difference between using 'int' and 'Integer' is explained in primitive and non-primitive types (tba).

Floating point numbers

A floating point number in BluePrint is a 32-bit floating point number, following the IEEE 754 standard. This means that a float can be a number between -3.40 * 10^38 and 3.40 * 10^38, and with ~7 decimal places.

An floating point number can be declared and defined as following:

# Declaring a float
float myFloat;

# Declaring and defining a float
float myFloat = 10.124356;

# Declaring a Float
Float myFloat;

# Declaring and defining a Float
Float myFloat = 10.124356;

The difference between using 'float' and 'Float' is explained in primitive and non-primitive types (tba).

Double precision numbers

A double precision number in BluePrint is a 64-bit floating point number, following the IEEE 754 standard. This means that a double can be a number between -1.80 * 10^308 and 1.80 * 10^308, and with ~15 decimal places.

An double precision number can be declared and defined as following:

# Declaring a double
double myDouble;

# Declaring and defining a double
double myDouble = 10.1234567890;

# Declaring a Double
Double myDouble;

# Declaring and defining a Double
Double myDouble = 10.1234567890;

The difference between using 'double' and 'Double' is explained in primitive and non-primitive types (tba).

Long integers

A long integer is a 64-bit integer. This means that a long integer can eb a whole number between -2^63 and 2^63 - 1.

A long integer can be declared and defined as following:

# Declaring a long
long myLong;

# Declaring and defining a long
long myLong = 12345678901234567890;

# Declaring a Long
Long myLong;

# Declaring and defining a Long
Long myLong = 12345678901234567890;

The difference between using 'long' and 'Long' is explained in primitive and non-primitive types (tba).
